# PPTX Export Handoff

## Goal

Use this file when the final delivery target includes `pptx` or `both`.

The purpose is to keep `ppt-as-code` HTML-first while making PPTX export predictable and stable.

## Product Boundary

- `ppt-as-code` is still the authoring skill for HTML decks.
- PPTX export is a post-process that starts only after the static HTML deck is ready.
- The supported input is a deck generated by `ppt-as-code`, not an arbitrary hand-written HTML page.

## Export Target Contract

Supported delivery targets:

- `html`
- `pptx`
- `both`

Default target is `html` unless the user explicitly asks for PPTX output.

## Required Handoff Artifacts

When the target includes PPTX, the handoff bundle must include:

- `index.html`
- `deck_manifest.json`
- `assets/`

Expected final output:

- `output.pptx`

## deck_manifest.json Contract

The manifest is the PPTX bridge and should be treated as the export source of truth.

Top-level fields:

- `deckTitle`
- `slideSize`
- `themeTokens`
- `slides`

Minimum per-slide fields:

- `id`
- `title`
- `exportModeHint`

Allowed export hint:

- `raster`

## Authoring-Side Responsibilities

Before handoff, `ppt-as-code` should:

1. define a stable export state for every slide
2. provide an export mode that hides:
   - controls
   - progress
   - slide number
   - debug overlays
3. ensure local assets are ready
4. avoid promising editable PPT output

## Rendering Contract

Use screenshot-only rendering:

- each slide is rendered from HTML as a stable screenshot
- each screenshot is placed into PPTX as a full-slide image

Do not use:

- native editable PPT reconstruction
- partial-raster mixed layouts
- DOM-to-PPT template rebuilding

## Motion Downgrade Rules

- PPTX export is static-only.
- Keep HTML motion in the HTML deliverable.
- Choose one stable slide state for PPTX output.

## Asset Rules

- Prefer local asset paths when image files exist.
- If an image asset is missing, do not silently drop it.
- If a slide cannot be captured correctly because of missing assets, stop with a clear missing-asset report.

## Companion Skill Handoff

Use `pptx-export-for-ppt-as-code` as the downstream export skill.

That companion skill should:

1. validate the manifest
2. enter export-safe browser mode
3. capture one screenshot per slide
4. place those screenshots into `output.pptx`
5. report capture success or blockers
